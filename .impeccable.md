# MyFav Design Context

## Users

个人收藏导航站，主要面向中文用户社区。用户可能是开发者或技术爱好者，需要一个整理自己收集的网站、GitHub 仓库和博客文章的工具。使用场景包括：

- 个人日常查找收藏的资源
- 向他人分享自己的收藏集
- 快速浏览和筛选内容

## Brand Personality

**均衡舒适**

- 简洁但不冷清
- 高效但不拥挤
- 专业但不严肃

界面应该让人感到轻松、有序、易于使用，而不是炫技或过度设计。

## Aesthetic Direction

### 视觉风格
- 干净简约的设计语言
- 适度的留白和间距
- 清晰的信息层级
- 中性色调为主，蓝色点缀

### 主题
- 支持亮色/暗色主题切换
- 自动检测系统偏好
- 主题切换流畅过渡

### 响应式
- 移动端优先适配
- 触控友好的交互区域

### 字体
- 正文：Noto Sans SC（中文）+ DM Sans（英文/数字）
- 清晰易读，不需要花哨

### 色彩
- 主色：蓝色系 (#2563eb)
- 中性灰色调用于文字和背景层次
- 标签使用柔和的背景色

### 参考风格
- Notion 的简洁感
- GitHub 仓库列表的信息密度
- Apple 官网的空间感

### 避免方向
- 过度装饰和动效
- 高饱和度的配色
- 信息过载的布局
- 只追求"酷炫"而牺牲可用性

## Design Principles

1. **内容优先** - 用户是来查找内容的，界面不应喧宾夺主
2. **简洁有序** - 清晰的分类和层级，让用户快速找到目标
3. **舒适阅读** - 合适的字号、行高、间距，长时间使用不疲劳
4. **灵活适配** - 在任何设备上都能良好使用
5. **无障碍** - 合理的对比度、可操作的交互区域
